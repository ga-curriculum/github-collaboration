**Document layout**

- [ ] All documents should start with an `H1` header written in title case (capitalizing all significant words in the header). There should never be another `H1` header in a document. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#formatting-documents).
- [ ] Student-facing documents should have a hero image for the document that immediately follows the opening `H1` header. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#formatting-documents).
- [ ] Microlesson content shouldn't have a table of contents. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#formatting-documents)
- [ ] `H2`-`H4` headers should be written in sentence case. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#formatting-documents).
- [ ] Header levels should not be skipped (an `H3` should not follow an `H1`, for example) and should not directly follow one another (add some written copy or an asset between them). [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#formatting-documents).
- [ ] You do sections should all have a header that reads as: 🧠 You do [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#you-do).
- [ ] Review question sections should all have a header that reads as: ❓ Review questions [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#review-questions).

**Text formatting**

- [ ] Variable names are placed in `inline code blocks`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#variables-or-short-lines-of-code).
- [ ] Short lines of code used in copy are placed in `inline code blocks`. Longer lines of code or multi-line blocks of code are placed in code blocks. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#variables-or-short-lines-of-code).
- [ ] File names are placed in `inline code blocks`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#directories-file-names-and-file-paths).
- [ ] Keyboard shortcuts are placed in `inline code blocks`. Keyboard shortcuts are provided for macOS, Windows, and Linux. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#keyboard-shortcuts).
- [ ] Raw URLs and URIs should be wrapped in `inline code blocks`. URLs should also include a link to that location. Raw URLs are not used without a clear reason and should typically have a text mask. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#urls-and-uris).
- [ ] Directories, file names, and paths should be wrapped in `inline code blocks`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#directories-file-names-and-file-paths).
- [ ] Property names should be written as `obj.property` or `property`. Methods should be written as `obj.method()` or `method()`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#properties-or-methods).
- [ ] When instructing students to type text - for example, to fill out a form - use `inline code blocks`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#ui-interactions).
- [ ] When giving directions for interacting with UI elements, use **bold text** to indicate the content of that element. For example, click the **Play** button. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#ui-interactions).
- [ ] New terms should be identified with *italicized text*. The term should also be *italicized* in the accompanying callout. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#terms-and-definitions).
- [ ] Adding emphasis is done by using ***bold and italicized text***. This should be done sparingly. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#adding-emphasis).

**Modularization and miscellaneous**

- [ ] Any list should have uniform ending punctuation. If any item in a list requires ending punctuation, all items should have ending punctuation. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#lists).
- [ ] Timeframes are not used in any content, for example, *this afternoon*, *yesterday*, *tomorrow*, etc. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#timeframes).
- [ ] Content outside the current module, including labs, should not be mentioned or linked to. You ***can*** reference other microlessons within a module. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#referencing-content).
- [ ] Any callouts should use the appropriate emojis. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#callouts).
- [ ] No placeholder text (**tktk**) is present. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#placeholder-text).
- [ ] Images should all have alt text. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#placeholder-text).
- [ ] The content is free of grammatical and spelling errors.

**Tone and voice**

- [ ] No Latin abbreviations. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#latin-abbreviations).
- [ ] No scare quotes. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#language-quirks).
- [ ] Avoid idioms and words that could be easily misunderstood. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#language-quirks).
- [ ] Avoid belittling language. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#avoiding-belittling-language).
- [ ] The AI assistant term is used in place of specific AI. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#ai-assistants).
- [ ] Features from ES versions older than the last three releases shouldn't be specifically called out. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#ecmascript-versions).
- [ ] Oxford commas are used to ensure clarity; double spaces are not used after punctuation. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#spacing).
- [ ] Avoid ampersands and ellipses in copy. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#ampersands).
- [ ] Titles (`H1`s) use AP Style title casing. All other content uses standard sentence casing. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#casing).
- [ ] General tone and voice aligns with the guidelines present in the [Technical Voice doc](./technical-voice.md) checklist.

**Markdown formatting**

- [ ] The full relative path is used to link to files within the project. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#relative-paths).
- [ ] One line break is present between markdown elements. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#line-breaks).
- [ ] Bold text is formatted with `**`; italicized text with `*`; bold and italicized text with `***`. [Style Guide](https://git.generalassemb.ly/modular-curriculum-all-courses/README/blob/main/style-guide/writing-style.md#bold-and-italicized-text).
